# 对话交接 / 进度日志

切换对话用。记录本轮所有改动、决策、SDK 限制和待办。

## 项目
VATSIM Companion：vPilot 插件(C#/.NET48) → Bridge(C#/.NET7) → cloudflared 隧道 → 手机 App(Flutter)。
把 vPilot 的频率/私信/广播/管制上下线/连接状态实时镜像到手机。

## 已确认的 SDK 硬限制（重要，别再尝试）
基于 `C:\Users\16832\AppData\Local\vPilot\Plugins\RossCarlson.Vatsim.Vpilot.Plugins.xml`：
- 插件 SDK **无法获取本机飞机状态**（位置/高度/速度/squawk 全无接口）。`AircraftUpdated` 是 sim 里**其他**飞机，不含本机。
- **METAR/ATIS** 只在插件主动调 `RequestMetar/RequestAtis` 后才触发，vPilot UI 里手动请求的不回调插件 → 已移除这两个订阅。
- 消息事件**不带 vPilot 时间戳**，去重只能用插件收取时刻。
- 拿不到：SERVER 欢迎语、voice server 状态、本机频率、Squawk、命令回显。
- 能拿到：私信/频率/广播消息、管制上下线(带坐标+频率)、连接/断开网络、呼号。

## 本轮已修复(更早)
- 插件去掉 METAR/ATIS 订阅(订阅/取消订阅/两个 handler 全删)。
- Bridge `PluginEventController.HandleMessageEvent` 反序列化加 `PropertyNameCaseInsensitive=true`（修手机消息灰气泡无字 bug，根因：插件发 camelCase，MessageDto 是 PascalCase，大小写敏感导致字段全 null）。

## 本轮新需求(用户提的5项 + Controllers栏)及决策
1. CF 隧道抖动 → **SignalR 保留 + App 加 5 秒 REST 拉取兜底**(已选)。
2. 同信息多次发送 → **插件端去重**：key=`from|content`，2 秒内相同则跳过(已选)。
3. aircraft 实时状态拿不到 → **改用 VATSIM 官方 datafeed**(已选)。
4. 管制上下线：不发 ATIS 上线；上下线都带频率；**频率补零 bug**(28.325 显示成 028.325，应 128.325)。
5. Controllers 栏：按距离过滤显示周围管制，分组 CENTER/APP-DEP/TOWER/GROUND/RAMP/CLR/ATIS。**未定位则不显示**(已选)。

## 架构决策
VATSIM datafeed (`https://data.vatsim.net/v3/vatsim-data.json`) 在 **Bridge 端**拉取，解决需求3+4+Controllers栏。
datafeed 的 `frequency` 已是正确字符串，绕过补零 bug(仅插件路径需修)。
本机呼号来自 `IPluginCommunicationService.GetCurrentStateAsync().Callsign`，用它查 datafeed 的 pilots[] 得到本机位置，再算周围管制。

## 已完成的代码改动(本轮)
### ⭐⭐⭐ 数据源回退官方 datafeed + 飞行相位精确判断 (最新，已编译 0 错误)
- **vatsim-radar 不可用**：`vatsim-radar.com/api/data/vatsim/data` 有 Cloudflare 防爬，对非浏览器请求返回 **403 + "Just a moment" 挑战页**(加 User-Agent 也没用)。已**改回官方 `https://data.vatsim.net/v3/vatsim-data.json`**(实测 590 pilots/47 ctrl/28 atis，无 CF)。
- `VatsimDataService.cs`：模型适配官方扁平结构(controllers[] + 独立 atis[]，无 vatsim-radar 的嵌套 controller 包装)。`GetControllersNear` 合并 controllers+atis，过滤 OBS(facility<=0 非 ATIS) 和占位频率 199.998。
- **飞行相位判断**(官方 datafeed 无 status 字段，自己推断)：
  - `VatsimPilot` 加 `flight_plan`(departure/arrival/altitude巡航高度)。
  - `VatsimDataService` 每轮拉取时快照上一轮各机高度 → `GetPreviousAltitude(callsign)`，用 15 秒高度差判升降趋势。
  - **机场坐标**用 `https://api.xflysim.com/pilot/api/realTimeMap/airports/{ICAO}` (返回 latitudeDeg/longitudeDeg/elevationFt)。`GetAirportAsync` 带缓存(查过的存 `_airportCache`，失败存 null 避免反复请求)。
  - `AtcController.InferStatus`(async)：综合 速度+高度+巡航高度+升降趋势+到出发/到达机场距离(Haversine)。状态值: atGate/taxiing/climbing/cruising/descending/approaching/arrived/enRoute。
- App `aircraft_status_screen.dart` `_formatStatus`：匹配上述 8 个状态值的友好显示。

### ⭐⭐ 消息去重 + 私信卡片式会话 (最新)
- **App 内消息重复**根因：`message_provider.dart` 的 `addMessage` 无去重，SignalR 实时推送 + `loadHistory` REST 加载同一条消息会重复。已加去重：相同 id，或 `from+content+type+10秒内时间戳` 视为重复则跳过。(插件端去重是上游，这是 App 端下游兜底)
- **私信回复报 "Please select a recipient"**：原 chat_screen 私信 tab 直接用底部输入框发送但无收件人。改为**卡片式会话**：
  - 新增 `screens/chat/private_chat_screen.dart`：与单个 peer 的会话页，顶部输入框可回复(调 `sendPrivateMessage(peer, text)`)。
  - `chat_screen.dart` 私信 tab 改为 `_buildPrivateConversations`：按对方呼号分组成会话卡片(显示最后一条消息+时间)，点卡片 Navigator.push 进 PrivateChatScreen。
  - 底部输入框/快捷指令**仅 Radio tab 显示**(`_tabController.index==0`)，加 tabController listener 切 tab 时 setState 刷新显隐。
  - `_sendMessage` 简化：移除私信分支，只发 radio。
  - 注: `_buildMessageBubble` 的 `isOutgoing=false` 占位是改动前就有的(产生 dead_code lint，仅 Radio tab 用，radio 都是收到的，合理)。

### ⭐ 数据源切换为 VATSIM-RADAR (已编译验证 0 错误)
