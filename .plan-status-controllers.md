# 实现计划：状态栏 + 管制列表 + 多项修复

## 背景与关键发现

调查 vPilot SDK 文档（`RossCarlson.Vatsim.Vpilot.Plugins.xml`）确认了一个**决定性事实**：

- IBroker **没有任何获取本机飞机状态的接口**（无位置/高度/速度/squawk）。`AircraftUpdated` 事件指的是 sim 里注入的**其他**飞机，不含本机。
- 因此**需求3（aircraft 实时状态）和"按距离过滤管制"无法通过插件实现**。

**决定（已与用户确认）**：改用 **VATSIM 官方 datafeed**（`https://data.vatsim.net/v3/vatsim-data.json`）作为权威数据源。一个 JSON 同时提供：
- `pilots[]`：用本机呼号查到自己的 `latitude/longitude/altitude/groundspeed/heading/transponder`
- `controllers[]`：全网管制 `callsign/frequency(字符串)/facility(int)/latitude/longitude/text_atis`
- `atis[]`：ATIS 席位
- `facilities[]`：facility id→类型映射（0=OBS,1=FSS,2=DEL,3=GND,4=TWR,5=APP,6=CTR）

datafeed 在 **Bridge 端**拉取（有公网、可缓存、避免每手机各自拉）。datafeed 的 `frequency` 已是正确字符串（如 "128.325"），天然绕过频率补零 bug。

> 实现时需对照真实 datafeed JSON 校验字段名（WebFetch 当时分类器故障未取到）。

## 五项改动

### 需求3 + Controllers栏：Bridge 接入 datafeed（核心）

**新增** `bridge-service/.../Services/VatsimDataService.cs`：
- `IHostedService` 后台服务，每 15 秒拉一次 datafeed（datafeed 本身约 15s 更新，再快无意义），缓存到内存。
- 暴露 `GetPilot(callsign)`、`GetControllersNear(lat, lon, rangeNm)` 方法。
- 距离用 great-circle（Haversine）计算。CTR/FSS 放宽到 600nm，APP/TWR/GND/DEL 用 150nm（facility 区分范围）。

**新增** `Controllers/VatsimController.cs`（或扩展 StatusController）：
- `GET /api/aircraft/state?callsign=XXX` → 从 datafeed 的 pilots 查本机，返回现有 `AircraftStateDto` 结构（填充真实位置/高度/速度/squawk）。
- `GET /api/atc/online?callsign=XXX` → 返回本机周围管制列表，按 facility 分组，每条含 callsign/frequency/type/atis。**若 datafeed 中查不到本机位置（未连线或未收录），返回空列表 + `located:false`，App 显示"等待定位"，不显示管制**（已与用户确认）。

**Program.cs**：注册 `VatsimDataService` 为 HostedService + 单例，注入 HttpClient。

> 本机呼号从哪来：插件 NetworkConnected 时已上报 callsign 并缓存在 Bridge（state 事件）。datafeed 查询用这个 callsign。

### 需求4：频率格式 + 管制上下线显示

**两条路径分别处理**：
- **datafeed 路径**（Controllers 栏）：frequency 直接用 datafeed 字符串，无需修复。ATIS 席位（`atis[]` / callsign 含 `_ATIS`）单独分组，不混入管制上线提示。
- **插件路径**（管制上下线系统消息，`Plugin.cs`）：修 `FormatFrequency`（[Plugin.cs:688](vatsim-companion-app/vpilot-plugin/VatsimCompanionPlugin/Plugin.cs#L688)）。
  - 当前 bug：`28325` → PadLeft补0 → `028.325`。
  - 修复：VATSIM 频率均为 1xx.xxx MHz，值 < 100000（即不足 6 位）时首位补 `1` 而非 `0`。`28325` → `128325` → `128.325`。
  - `OnControllerAdded`/`OnControllerDeleted`（[Plugin.cs:230-248](vatsim-companion-app/vpilot-plugin/VatsimCompanionPlugin/Plugin.cs#L230-L248)）：跳过 callsign 含 `_ATIS` 的席位（不发 ATIS 上线提示）；上线和下线都带频率（下线事件 SDK 只给 callsign，无频率——需在 OnControllerAdded 时缓存 callsign→freq，下线时查出来显示）。

### 需求2：去重（插件端，收取时刻+内容）

`Plugin.cs` 消息 handler（Private/Radio/Broadcast）：维护一个小的最近消息缓存 `Dictionary<string,DateTime>`，key = `from|content`，收到时若 2 秒内有相同 key 则跳过不上报。轻量，无外部依赖。

### 需求1：App 5秒拉取兜底 + 自动重连

App 端，**保留 SignalR 实时推送**，额外加 5 秒定时拉取：
- `connection_provider.dart` / 新增轮询逻辑：每 5 秒调 `/api/aircraft/state` + `/api/atc/online`，更新 aircraftState 和 controllers provider。
- SignalR 已有 `withAutomaticReconnect()`；确认 `websocket_service.dart` 重连配置，必要时缩短重连间隔。
- 5 秒拉取保证即使 SignalR 因隧道抖动断开，状态/管制列表仍持续刷新。

### App UI：Status 页加 Controllers 栏

- 新增 `models/controller_info.dart`（callsign/frequency/type/atis）。
- 新增 `providers/controllers_provider.dart`（5 秒拉取 `/api/atc/online`）。
- `aircraft_status_screen.dart` 改成两区：上半 Aircraft Status（真实数据），下半 Controllers，按 CENTER/APP-DEP/TOWER/GROUND/RAMP/CLR/ATIS 分组显示，每条显示 callsign + frequency。未定位时下半区显示"等待定位…"占位，不显示管制。

## 部署（用户执行）
- Bridge：重新 `dotnet run`（datafeed 需 Bridge 联网）
- 插件：退出 vPilot → `dotnet build -c Release` → 重启 vPilot
- App：`flutter build apk --release` → 安装

## 注意
- datafeed 字段名实现时以真实 JSON 为准校验。
- Bridge 需能访问 `data.vatsim.net`（公网）。
- 本机状态依赖"本机已在 VATSIM 连接且 datafeed 已收录"，刚连上有 ~15-60s 延迟才出现在 datafeed。
